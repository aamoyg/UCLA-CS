9-1.    (a) A pointer is an object which contains the address of another
object. (p238)

    (b) Garbage collection is the process of recovering unused memory cells and
returning them to a free list or compacting all the memory in use to contiguous
locations and relcaiming the unused cells as contiguous storage. (p254)

    (c) A jump table is a table of consecutive jump instructions, such as at
the beginning of a library, which provides fixed, externally-known addresses to
subroutines while allowing changes to internal addresses without affecting the
interface. (p255)

    (d) Base addressing is an addressing mechanism which uses a pointer
register to contain the base address of a structure and uses offsets in
instructions to access elements within the structure. (p244)

    (e) A linked structure is a structure consisting of independent cells
linked together with pointers from one to the next. (p235)

    (f) Dynamic memory allocation is the process of allocating memory (usually
from a heap) as requested by program. (p250)

9-2.    The addressing of a variable by a pointer requires a dereferencing step
in which the pointer value, which is the actual address of the operand, is
obtained from the pointer and then used to access the operand.

9-3.    A linked list requires additional space with each data item to store
the links, but insertion and deletion are faster than moving items in the list
during insertion and deletion.

9-4.    Linked cells are connected by links, thus the location of each cell is
independent of the locations of other cells.  Cells have no links between
fields, thus the entire cell must be stored in consecutive memory locations.

9-6.    The FP and XR are both added to the address in the instruction.  The
only addressing modes where they operate differently are the indirect and the
indirect indexed modes where the FP is applied immediately to the address.
Thus if immediate, direct, frame direct, or indexed addressing is used, either
FP or XR may be used as a base register.

9-7.    A circular-type definition is one in which one part of the definition
references another part of the definition which occurs later.

9-8.    An unassigned pointer variable has random data while a pointer variable
with the value NIL contains an explicit value reserved to indicate that the
pointer points to nowhere (has not been set to a value).
