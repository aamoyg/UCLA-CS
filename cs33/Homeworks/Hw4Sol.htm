4-1.    (a) A location counter is an assembler counter which is used to
specify the location of the word currently being generated. (p84)

    (b) A symbol is an alphanumeric string used to give a name to a value,
usually a location. (p66)

    (c) Object code is assembler output which specifies the bit pattern which
is to be placed in each word of memory. (p88)

    (d) An assembler directive is an instruction to an assembler to control
its operation. (p77)

    (e) Forward referencing is the use of a symbol which is defined later in
the source code. (p84)

    (f) A symbol table is an assembler table which contains symbols which have
appeared in the assembly and, by the end of Pass 1, the value for each symbol.
(p85)

4-2.    (a) The .WORD and .BLKW directives will always change the location
counter by the number of words reserved  The .EQU directive will change the
location counter when it is specified as the first parameter (.EQU @,value).

    (b) The .WORD and .BLKW directives will allocate memory words as specified.

        The .PAGE directive will neither change the location counter or
allocate any memory.  It merely causes a page eject on the assembler listing.

4-3.    Neither Y or Z are defined at the time they are used.  This is an
example of illegal forward referencing which is not permitted in the .EQU
directive

4-4.    The only difference is in the manner of their definition.  Otherwise
they are the same.
