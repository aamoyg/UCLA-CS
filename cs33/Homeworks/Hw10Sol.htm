10-1.(a)    Mutual recursion is a condition in which two or more functions are
defined in terms of each other.  For example, Function A calls Function B as a
part of its definition and Function B also calls Function A as part of its
definition. (p271)

    (b) The state of a program consists of all information which defines the
progress of the execution of a program up to a specified point. (p274)

    (c) Local variables are variables which are only used inside a program and
are not available outside the program. (p274)

    (d) A reentrant program is a program which can be interrupted at any point,
called again during the interrupt, and then be resumed after the interrupt and
operate correctly. (p303)

    (e) Performance measurement is the process of analyzing a program to
determine its speed and memory requirements and, optionally, its use of other
resources. (p293)

    (f) Tail recursion optimization is the process of converting a recursive
program into an iterative program by changing the self-call to an internal
loop. (p302)
 
 

10-2.(a)    The only property required to allow the application of tail
recursion optimization is that a recursive call be the last operation performed
by the subroutine other than returning to the calling program.

    (b) The steps to optimize a subroutine to its tail recursive solution are
the following:

1.  Copy the actual arguments of the recursive call into the formal argument
positions of the previous call in the stack.

2.  Remove the ADS instruction which adjusts the stack pointer after the
recursive call.

3.  Change the starting location of the iteration from the first instruction in
the program to the first instruction after the call initialization (the BGN
instruction).

4.  Change the JSR to a JMP to the location which was determined in Step 3
above.
 

10-3.(a)    Recursive subroutines are usually reentrant because all local
storage is usually on the stack and thus will be unique for each call.

    (b) A recursive subroutine will not be reentrant if it is not data
consistent at all times.

10-4.   Show that if the program is interrupted at any place, the value of the
variable for the interrupted program cannot by changed by the interrupting
program.

10-5.   By creating variables on the stack, each instance of a program obtains
a different set of memory locations (on the stack) for its copy of the local
variables.
