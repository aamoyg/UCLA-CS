6-1.(a) A LIFO stack is a last-in, first-out stack which returns values in the
reverse order from entry:  the last value pushed onto the stack is the first
value returned, the next-to-last value pushed onto the stack is the second
value returned, etc. (p133)

    (b) An nested call is a subroutine call which is itself located inside a
subroutine. (p143)

    (c) Stack overflow is a condition which occurs when a stack "grows" to such
a size that it exceeds the size allocated to it and overwrites memory allocated
to other data or instructions. (p136)

    (d) A subroutine call is a transfer of control to another program, possibly
with parameter information specified, with a return to the instruction after
the call in the calling program when the subroutine terminates. (p142)

    (e) Indirect addressing is the use of a memory word as the source of the
actual address of an operand. (p156)

    (f) A stack pointer is a pointer to the top word of a pushdown stack. (p134)

    (g) A calling convention is an established procedure or convention used by
both a calling program and a called program to pass parameters from the calling
program to the called program. (p148)

    (h) A stack frame is a set of (consecutive) locations in a stack containing
the parameters, saved data/registers, and other data associated with a specific
program         or subroutine. (p152)

6-2.CBA:    Call By Address:  The address of a parameter is passed on the stack.
    CBV:    Call By Value:    The value of a parameter is passed on the stack.

CBV is appropriate for single values which are not to be changed in the calling
program.  CBA is used for arrays and for parameters which are to receive values
from the called program.
 
 

6-3.    Shared Memory:  Fixed memory addresses are used for parameter passing.
            Advantage:      Simple, reasonably fast.
            Disadvantages:  Programs tightly coupled;
                            Subroutine cannot be reused because of shared
                            variable name conflicts;
                            Shared variables may be accidentally modified by
                            another program.

        Registers:  Hardware registers are used for parameter passing.
            Advantages:     Simple;
                            Fast.
            Disadvantages:  Limited number of registers;
                            Cannot transfer large amounts of data;
                            Variables may be accidentally modified.

        Stack:  The machine stack is use for parameter passing.
            Advantage:      Avoids problems of other methods.
            Disadvantages:  Complex;
                            Slower;
                            Less efficient.

6-4.    Formal Parameters:  Placeholders for parameters in a called program.
        Actual Parameters:  Actual parameters passed by a calling program.

The formal parameters specify the relative location in the stack where each
parameter will be placed.  The actual parameters are the values and addresses
which are placed in the stack before the calling program makes the call to the
called program.

6-5.    If registers were not saved during a subroutine call, any values in the
registers might be changed during the execution of the subroutine and not
restored to their values at the time of the call when control is passed back to
the calling program from the called program.  The calling program would
continue to execute expecting the same values in the registers as before the
call and would calculate incorrect results (or worse).  Registers can be saved
and restored either by the calling program of the called program.

6-6.    Frame addressing can be used to facilitate accessing values in the
stack which are not at the top of the stack.  For example, LDA ! 2 accesses the
third word on the stack if the frame pointer had been set to the stack pointer
at the time of the subroutine call.  Indirect addressing (combined with frame
addressing) supports the CBA mechanism so that a parameter may be the address
of a variable parameter.  For example, LDA* ! 2 accesses the third word on the
stack as the address of a variable which holds the value of the parameter.
Indirect indexed addressing (combined with frame addressing) adds the
additional capability of utilizing an offset from a base address specified in
the stack. For example, LDA& ! 2 accesses the third word in the stack as the
base address of an array and then adds the value in the index register to
obtain the address of the word which holds the value of the parameter.
